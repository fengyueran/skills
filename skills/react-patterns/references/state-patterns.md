# 状态模式

用于判断状态应该放在哪一层，以及该不该共享。

## 目录

- 1. 本地状态
- 2. 提升状态
- 3. Context
- 4. 外部 store
- 5. 服务端状态

## 1. 本地状态

默认首选。

适用：

- 只被当前组件使用
- 生命周期和当前组件一致
- 不需要跨树共享

## 2. 提升状态

适用：

- 两个或多个兄弟组件共享同一状态
- 共享范围仍然很小

原则：

- 只提升到真正的最近公共父级
- 不要因为“以后可能会用”而过早提升

## 3. Context

适用：

- 一段子树需要共享同类状态
- 不想层层 props 透传

不适用：

- 高频变化且消费者很多的状态
- 可以通过简单 props 解决的问题

## 4. 外部 store

适用：

- 跨页面或跨模块共享
- 业务状态复杂
- 需要统一读写入口或中间件能力

常见选择：

- Zustand
- Redux Toolkit

## 5. 服务端状态

不要和本地 UI 状态混为一谈。

适用：

- 请求结果
- 缓存
- 重试、失效、重新拉取

优先考虑：

- React Query
- SWR
