# Hook 模式

用于判断何时提取 custom hook，以及如何控制抽取边界。

## 目录

- 1. 何时提取 custom hook
- 2. 何时不要提取
- 3. 常见适用场景

## 1. 何时提取 custom hook

适用：

- 多个组件复用同一段状态逻辑
- 组件内有较长的副作用和状态编排
- 逻辑层已经可以独立于具体 UI 存在

典型场景：

- 请求与缓存
- 表单状态
- debounce / throttle
- 本地存储同步
- 订阅浏览器 API

## 2. 何时不要提取

- 逻辑只在一个很小的组件里出现一次
- 抽出来只会让命名和跳转成本上升
- hook 返回的数据结构比原组件还难理解

## 3. 常见适用场景

- `useFetch`：封装请求生命周期
- `useDebounce`：管理输入节流与延迟
- `useLocalStorage`：同步持久化值
- `useForm`：封装表单状态和校验

原则：

- hook 负责逻辑复用
- 组件负责表达 UI
- 不要把 UI 细节塞回 hook
